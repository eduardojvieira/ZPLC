# Runtime & Embedded

The ZPLC Runtime is a deterministic, highly portable C99 Virtual Machine designed to execute `.zplc` bytecode on resource-constrained embedded devices.

## VM Model

The core of ZPLC is a stack-based virtual machine. It interprets instructions generated by the compiler.
*   **Determinism**: The VM is designed to execute instructions with highly predictable timing, a critical requirement for industrial control.
*   **No Dynamic Memory**: After initialization, the core VM performs zero dynamic memory allocations (`malloc`, `free`). All memory is statically allocated or managed via pre-allocated pools, eliminating memory fragmentation risks during long-running operations.

## Scheduler and Execution Model

ZPLC uses deterministic task execution within the context of the underlying runtime
environment (usually Zephyr, but also POSIX and WASM for validation paths).

*   **Tasks**: A PLC program is composed of one or more tasks.
*   **Cycle Time**: Each task is assigned a specific cycle time (e.g., 10ms, 100ms).
*   **Execution Loop**: 
    1.  **Read Inputs**: The HAL reads the current state of physical inputs.
    2.  **Execute Logic**: The VM executes the bytecode for the task.
    3.  **Write Outputs**: The HAL writes the new state to the physical outputs.
    4.  **Wait**: The scheduler yields control until the next cycle begins.

## Supported Targets

The runtime is completely decoupled from the hardware via the HAL. It can run on any platform where a HAL implementation exists.

Validation targets in v1.5:

- **Zephyr RTOS**: primary embedded execution target
- **POSIX**: host-side compile and simulation validation
- **WASM**: IDE simulation path

Concrete board claims are not defined here. They come from the supported-board manifest and
the reference section.
